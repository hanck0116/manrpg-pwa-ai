## 2. 최초 캐릭터 생성 방법

1. 첫 화면의 캐릭터 생성 패널에서 힘/민첩/체력/지능/지혜/외모를 조정합니다.
2. 남은 포인트가 0이 되면 생성 완료를 누릅니다.
3. 플레이어 1명, 적 1명, 고정 7x7 맵인지 확인합니다.

## 3. 기본 전투 테스트 순서

1. 이동 행동을 큐에 추가합니다.
2. 기본 공격을 큐에 추가합니다.
3. 턴 마무리를 누릅니다.
4. 로그에 행동 순서와 피해가 기록되는지 확인합니다.

## 4. 적 처치 후 층 클리어 테스트

1. 실험/디버그 패널에서 적 HP 1로 만들기를 누릅니다.
2. 기본 공격으로 적을 처치합니다.
3. phase가 층 클리어로 바뀌는지 확인합니다.

## 5. 보상 선택 테스트

1. 층 클리어 회복/정비를 누릅니다.
2. 보상 후보에서 선택 가능한 수만큼 보상을 선택합니다.
3. 보상 확정을 누르고 코인/인벤토리 반영을 확인합니다.

## 6. 레벨업 스탯 분배 테스트

1. 보상 확정 후 레벨업 스탯 분배 패널을 확인합니다.
2. 남은 포인트가 0이 되도록 스탯을 분배합니다.
3. 레벨업 분배 완료 후 다음 층 진입이 가능한지 확인합니다.

## 7. 인벤토리 사용/판매 테스트

1. 실험/디버그 패널에서 테스트용 보상 아이템 지급을 누릅니다.
2. 정비 단계에서 외공서/내공서/검기/마법서를 사용합니다.
3. 판매 버튼으로 코인이 증가하는지 확인합니다.

## 8. 상점 구매 테스트

1. 실험/디버그 패널에서 테스트용 코인 +50을 누릅니다.
2. 정비 단계의 상점에서 아이템을 구매합니다.
3. 코인 감소와 인벤토리 추가를 확인합니다.

## 9. 마법서 습득 테스트

1. 기초 마법서를 인벤토리에 준비합니다.
2. 정비 단계에서 사용을 누릅니다.
3. 성공 시 보유 마법에 추가되고, 실패 시 마법서가 유지되는지 확인합니다.

## 10. magicTicket random 테스트

1. 기초 마법서 뽑기권을 인벤토리에 준비합니다.
2. 정비 단계에서 사용을 누릅니다.
3. 판정 없이 등급 범위의 마법 1개가 보유 마법에 추가되고, 뽑기권이 제거되는지 확인합니다.

## 11. magicTicket select 테스트

1. 실험/디버그 패널에서 테스트용 보상 아이템 지급을 누릅니다.
2. 기초 마법서 선택권 또는 중급 마법서 선택권을 사용합니다.
3. 선택권 사용 패널이 표시되는지 확인합니다.
4. 마법을 선택하면 보유 마법에 추가되고, 선택권이 인벤토리에서 제거되는지 확인합니다.
5. 취소를 누르면 보유 마법은 늘지 않고 선택권이 유지되는지 확인합니다.

## 12. choice 선택권 테스트

1. choice 타입 아이템이 있는 저장 상태를 준비합니다.
2. 선택지가 있는 choice 아이템을 사용하면 선택권 사용 패널이 표시되는지 확인합니다.
3. 선택 확정 시 실제 효과가 적용되지 않고 로그만 남으며 인벤토리가 유지되는지 확인합니다.

## 13. 마법 시전 테스트

1. 보유 마법 패널에서 마법을 큐에 추가합니다.
2. 턴 마무리를 눌러 로컬 TypeScript 전투 처리를 확인합니다.
3. AI가 HP/MP/피해/위치/스탯을 바꾸지 않는지 확인합니다.

## 14. 반응턴 테스트

1. 적 공격으로 반응턴이 발생하는 상황을 만듭니다.
2. 회피/방어/카운터/반응 안 함을 각각 눌러 봅니다.
3. 로그와 phase 복귀를 확인합니다.

## 15. AI 연결 테스트

1. AI 설정에서 Provider와 API 키를 입력합니다.
2. AI 사용을 켜고 연결 테스트를 누릅니다.
3. 마지막 연결 테스트 결과와 fallback 상태를 확인합니다.

## 16. Worker proxy 테스트

1. Worker `/health` 주소를 브라우저 또는 curl로 열어 `ok: true`를 확인합니다.
2. AI 설정에서 Worker 프록시 사용을 체크합니다.
3. Worker URL에 배포된 Worker 주소를 입력합니다.
4. 연결 테스트를 눌러 프록시 연결 결과와 fallback 상태를 확인합니다.
5. 같은 Provider/API 키로 Direct BYOK와 Worker Proxy를 각각 테스트합니다.
6. 실패해도 게임 진행이 멈추지 않고 fallback 로그가 남는지 확인합니다.

## 17. 저장/불러오기 테스트

1. 저장을 누릅니다.
2. 새로고침 후 불러오기를 누릅니다.
3. phase, 인벤토리, 보유 마법, pendingChoice, 로그가 검증된 상태로 복구되는지 확인합니다.

## 18. 디버그 패널 사용법

실험/디버그 패널은 테스트 편의 기능입니다. 저장 초기화, 새 캐릭터 초기화, JSON 복사, 테스트 코인/아이템/마법 지급, 적 HP 1 설정, 플레이어 HP/MP 회복을 제공합니다.
