# ManRPG 규칙 추출 현황

## 구현 완료: 스킬 실제 효과 1차

원본 통합본에서 확인한 스킬 관련 확정 규칙은 스킬명, 공격 피해 배율, 판정 보정, MP/HP 변화, 스킬 초기화 시 `스킬목록` 제거, 그리고 "스킬 피해 배수를 최종 공격 피해에 곱했는가?"입니다. 세부 산식이 확정되지 않은 부분은 임의로 확장하지 않습니다.

- 현재 저장 구조는 `saveVersion: 11`입니다.
- `GameState.skills`에 보유 스킬을 저장합니다.
- `QueuedAction.skillId`로 행동 큐의 스킬 사용 대상을 연결합니다.
- 메인 타이밍 스킬만 플레이어 메인 턴에 사용할 수 있습니다.
- 피해 스킬은 1차 임시 산식으로 계열별 기준값에 배율을 곱해 적 1명에게 적용합니다.
- 회복 스킬은 1차 임시 산식으로 플레이어 1명의 HP를 최대 HP 이하로 회복합니다.
- 방어 스킬은 플레이어 방어 상태만 켭니다.
- 특수 효과, 상태 이상, 반응 스킬 연결, 광역 효과는 TODO로 남깁니다.
- 스킬 초기화권은 보유 스킬 목록만 비우고 다른 상태는 건드리지 않습니다.

유지 조건: 플레이어 1명, 적 1명, 보스 없음, 고정 7x7 맵, AI/Worker의 규칙 계산 금지.

## 고정 전투 구조

- 플레이어는 항상 1명입니다.
- 적은 항상 1명입니다.
- 보스몹은 생성하지 않습니다.
- 맵은 항상 동일한 7x7 고정 맵입니다.
- 파티, 3인 플레이어 목록, 플레이어 선택, 플레이어 교체 구조는 사용하지 않습니다.
- AI는 판정에 관여하지 않습니다.
- 규칙·판정·전투·보상·미니맵은 로컬 TypeScript에서 처리합니다.
- AI는 GM 묘사, 애매한 행동 해석, 희소한 설명 생성에만 사용합니다.
- AI API 1차 구현은 브라우저 직접 BYOK 방식입니다. API 키는 기본적으로 메모리에만 두며, 사용자가 명시적으로 체크한 경우에만 localStorage에 저장합니다.
- AI prompt/response/API 키는 저장 데이터에 포함하지 않습니다. 현재 저장 구조는 `saveVersion: 10`입니다.

## 구현 완료: 캐릭터 생성/스탯 분배

이번 단계에서는 1인 플레이어의 최초 캐릭터 생성 UI를 구현했습니다. 플레이어는 항상 1명이며, 적은 항상 1명이고, 보스몹/파티/플레이어 선택/3인 구조는 만들지 않습니다.

- `GameState.setupMode`
  - `true`: 캐릭터 생성/스탯 분배 중
  - `false`: 전투 진행 중
- 시작 레벨: `1`
- 기본 스탯: 힘/민첩/체력/지능/지혜/외모 각각 `1`
- 추가 분배 포인트: `54`
- 총 시작 스탯 합계: `60`
- 조정 가능 스탯: 힘/민첩/체력/지능/지혜/외모
- 조정하지 않는 값: 외공/내공/검기/멀티캐스팅/특성/코인
- 총 스탯 포인트 총량: `60 + (level - 1) * 3`
- 사용 스탯 포인트: 힘 + 민첩 + 체력 + 지능 + 지혜 + 외모
- 남은 포인트: 총 스탯 포인트 총량 - 사용 스탯 포인트
- 스탯 최소값: `1`
- 스탯 최대값
  - 레벨 80 미만: `level + 20`
  - 레벨 80 이상: `100`
- 남은 포인트가 `0`일 때만 생성 완료할 수 있습니다.
- 생성 완료 전에는 이동/공격/스킬/마법/아이템/방어/대기/턴 마무리와 적 자동 턴을 진행하지 않습니다.
- 레벨업 UI는 `level-up-pending` 단계에서 구현되어 있으며, 최초 생성용 `setupMode`와 분리됩니다.

## 구현 완료: 최종 파생 수치 규칙 1차

`src/rules/derivedStats.ts`는 통합본의 `calc()`와 `KI` 테이블을 기준으로 다음 값을 계산합니다.

- `totalStatPoint = 60 + (level - 1) * 3`
- `usedStatPoint = strength + dexterity + constitution + intelligence + wisdom + appearance`
- `remainingStatPoint = totalStatPoint - usedStatPoint`
- `maxStat = level >= 80 ? 100 : level + 20`
- `baseHP = constitution * 10`
- `baseMP = level * 5 + intelligence * 10`
- `baseRegen = level + wisdom * 2`
- `basicAtk = floor((strength + constitution) / 10) + 2`
- `outerMultiplier = 1.4 ^ outerStack`
- `innerMultiplier = 1.2 ^ innerStack`
- `maxHP = floor(baseHP * outerMultiplier)`, 최소 1
- `maxMP = floor(baseMP * innerMultiplier + swordKiMpDelta)`, 최소 0
- `mpRegen = floor(baseRegen * innerMultiplier)`, 최소 0
- `attack = floor(basicAtk * swordKiAtkMul * attackTraitMultiplier)`, 최소 0
- `multi = multicasting * 2` if `traits` includes `시분할`, otherwise `multicasting`

### 검기 테이블

| 단계 | 이름 | 공격 배수 | MP 변화 |
| --- | --- | ---: | ---: |
| 0 | 없음 | 1 | 0 |
| 1 | 검기상인 | 1.5 | -50 |
| 2 | 검기 | 2 | -150 |
| 3 | 검사 | 5 | -300 |
| 4 | 검강 | 20 | -700 |
| 5 | 강기압환 | 50 | -500 |
| 6 | 심검 | 50 | 1,000,000 |

### 구현한 공격 특성 배수

- `연혼염`: x1.15
- `흑염의 영체화`: x1.1
- `헤일로`: x1.25
- `입천`: x1.3
- `앙케 라`: x1.5

## 구현 완료: 일반 판정/절대판정 정리

- 일반 판정 `rollStatCheck(effectiveStat, bonus)`
  - 원본 통합본에 명시된 `1d[유효스탯]`을 굴립니다.
  - 반환값은 `type`, `sides`, `roll`, `bonus`, `total`, `success?`, `note`입니다.
  - 임의 성공조건은 제거했습니다.
  - `success`는 기본적으로 `undefined`이며, 상황별 호출 규칙이 성공조건을 결정해야 합니다.
- 절대 판정 `rollAbsoluteCheck(effectiveStat, bonus)`
  - `1d100 <= effectiveStat + bonus`입니다.
  - 반환값은 `type: 'absolute'`, `roll`, `target`, `success`입니다.

## 구현 완료: 기본 공격 계산 정리

- 기본 공격은 인접 대상에게만 적용합니다.
- 기본 공격 피해는 `derived.attack`을 사용합니다.
- `derived.attack`은 검기와 구현된 공격 특성 배수를 반영한 최종 공격력입니다.
- 기본 공격 추가타는 공격자 민첩이 대상 민첩의 2배 이상일 때만 1회 발생합니다.
- 추가타는 기본 공격에만 적용합니다.
- 결과에는 `hits`, `damagePerHit`, `totalDamage`, 호환용 `damage`가 포함됩니다.

## 현재 전투 루프 구조

- 전투 시작 시 플레이어와 적의 민첩을 비교합니다.
- 더 높은 쪽이 선턴을 가져갑니다.
- 민첩이 같으면 플레이어가 선턴입니다.
- 플레이어 메인턴에서는 행동을 즉시 실행하지 않고 행동 큐에 추가합니다.
- 행동은 플레이어 메인턴에서만 추가할 수 있습니다.
- `battle-ended`, `floor-cleared`, `reward-pending` 상태에서는 행동을 추가할 수 없습니다.
- 적이 이미 쓰러졌으면 기본 공격 행동을 추가할 수 없습니다.
- 플레이어가 쓰러졌으면 행동을 추가할 수 없습니다.
- 턴 마무리 시 행동 큐가 순서대로 실행됩니다.
- 각 행동 실행 전후로 전투 종료 여부를 확인합니다.
- 적 HP가 0이 되면 남은 큐를 실행하지 않고 `floor-cleared` 승리 처리합니다.
- 플레이어 HP가 0이 되면 `battle-ended` 패배 처리합니다.
- 이동 중 벽/점유 칸에 막히면 해당 이동 행동은 중단하고 다음 행동으로 넘어갑니다.
- 플레이어 행동 큐 실행 후 플레이어 MP가 회복됩니다.
- 적이 살아 있으면 적 메인턴이 자동 진행됩니다.
- 적 메인턴에서는 임시 AI가 이동 또는 기본 공격을 실행합니다.
- 적 턴 종료 후 자동으로 플레이어 메인턴으로 돌아옵니다.

## 이동 임시 규칙

- 상하좌우 이동이 가능합니다.
- 이동 칸 수는 UI에서 1~3칸을 선택합니다.
- 벽으로 이동할 수 없습니다.
- 맵 밖으로 이동할 수 없습니다.
- 적이 있는 칸으로 플레이어가 이동할 수 없습니다.
- 플레이어가 있는 칸으로 적이 이동할 수 없습니다.
- 이동 실패 시 로그를 남깁니다.

1. 층 클리어 회복/상태 초기화
2. 5Lv 플러스 적용
3. 파생 수치 재계산
4. 보상 뽑기
5. 보상 선택
6. 레벨업 스탯 분배

구현 상태:

## 구현 완료: 보상 아이템 사용 효과와 마법서 습득 판정

원본 통합본 기준으로 코드화한 항목은 다음과 같습니다.

### makeItem() 판매가/타입

- 외공서: `type=martial`, 판매가 5코인
- 내공서: `type=martial`, 판매가 5코인
- 검기: `type=martial`, 판매가 12코인
- 멀티캐스팅의 서: `type=multi`, 판매가 10코인
- 스킬 초기화권: `type=reset`, 판매가 5코인
- 기초 마법서: `grade=기초`, 판매가 3코인
- 중급 마법서: `grade=중급`, 판매가 5코인
- 고급 마법서: `grade=고급`, 판매가 8코인
- 마도서: `grade=마도서`, 판매가 15코인

### 마법서 등급/서클/난이도

| 등급 | 서클 범위 | 습득 난이도 |
| --- | --- | --- |
| 기초 | 1~2서클 | 50 |
| 중급 | 3~4서클 | 70 |
| 고급 | 5~6서클 | 100 |
| 마도서 | 7~9서클 | 100 |

습득 판정은 `지혜 >= 난이도`이면 자동 성공입니다. 자동 성공이 아니면 `1d난이도 < 지혜`일 때 성공합니다. 실패해도 마법서는 유지되고 보유 마법에는 추가하지 않으며, 다시 시도할 수 있습니다.

### 구현한 사용 효과

- 외공서: 플레이어 1명의 `outerStack +1`, 파생 수치 재계산, 아이템 제거
- 내공서: 플레이어 1명의 `innerStack +1`, 파생 수치 재계산, 아이템 제거
- 검기: `swordKi +1`, 최대 6 제한, 성공 시 파생 수치 재계산 및 아이템 제거
- 멀티캐스팅의 서: `multicasting +1`, 파생 수치 재계산, 아이템 제거
- 마법서: 성공 시 `spells`에 마법명/서클/등급 추가 및 아이템 제거, 실패 시 아이템 유지
- 판매: `sell > 0`이면 코인 증가 및 아이템 제거, `sell=0`이면 판매 불가

## 구현 완료: 반응행동 1차

- 적이 인접 기본 공격을 시도하면 즉시 피해를 적용하지 않고 `player-reaction` phase와 `pendingReaction`을 설정합니다.
- 회피는 민첩 절대 판정, 방어는 힘 절대 판정, 카운터는 민첩 절대 판정을 사용합니다.
- 회피 성공은 공격 무효, 실패는 원 피해 적용입니다.
- 방어 성공은 공격 무효, 실패는 피해 1 감소 후 적용입니다.
- 카운터 성공은 공격 무효 후 플레이어 기본 공격 1회, 실패는 원 피해 적용입니다.
- 반응 안 함은 원 피해를 그대로 적용합니다.
- 반응은 턴을 소모하지 않으며 처리 후 플레이어 메인으로 돌아갑니다. 반응 중 적 HP가 0이면 `floor-cleared`, 플레이어 HP가 0이면 `battle-ended`가 됩니다.
- 플레이어의 방어 행동으로 생긴 `player.guarding=true`는 적의 다음 공격 1회 판정까지 유지되며, 공격/반응 처리 후 해제됩니다. 적이 이동만 하고 공격하지 못한 경우에는 플레이어 메인턴 복귀 시 해제됩니다.
